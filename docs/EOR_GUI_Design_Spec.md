# Eagle of Rome GUI 界面设计说明书

## 设计概述

基于 Eagle of Rome 已开发功能（MVP 0.7）设计的 GUI 界面原型。采用**统一布局框架 + 阶段内容动态切换**的设计策略，确保玩家在不同游戏阶段中拥有稳定的导航心智模型，同时主内容区域根据当前阶段展示对应的交互界面。

## 设计原则

1. **统一布局框架**：所有 7 个阶段共享相同的左-中-右三栏布局，降低学习成本
2. **信息层级清晰**：高频信息（国库、影响力、人物）始终可见，低频信息（事件日志、操作按钮）按需展示
3. **操作区域集中**：所有可执行操作集中在右侧面板，避免玩家在不同区域间频繁移动视线
4. **罗马风格配色**：深红 + 金 + 羊皮纸色，营造罗马共和国氛围

## 一、全局布局框架（4 区域结构）

```
+-------------------------------------------------------------+
|  [HEADER] 标题栏 + 回合信息 + 阶段进度条 [1/7]                |  <- 8% 高度
+--------+----------------------------+-----------------------+
|        |                            |                       |
|  LEFT  |         CENTER             |        RIGHT          |
|  20%   |         50%                |        21%            |
|        |      (阶段主内容区)         |      (操作+日志)       |
|        |                            |                       |
+--------+----------------------------+-----------------------+
|  [BOTTOM] 快捷信息栏 + 系统按钮                              |  <- 4% 高度
+-------------------------------------------------------------+
```

### 1. 顶部标题栏（Header Bar）
- **左侧**：游戏 Logo `[R] EAGLE OF ROME`
- **中部**：回合信息 `Round 1 (282 BC)`
- **右侧**：阶段进度条，7 个阶段以小方块排列，当前阶段高亮（金色），已完成阶段（棕色），未进行阶段（浅灰）
- **视觉**：深红色背景 + 金色文字，营造元老院庄严感

### 2. 左侧信息栏（Left Panel）- 持久面板

此面板在所有阶段保持不变，展示玩家需要**持续监控**的核心信息。

#### 2.1 国家状态（State Status）
- 国库资金：`Treasury: 142 Talents`
- 公地总量：`Public Land: 6000 C`
- 行省公地：`Provincial Land: 0`
- 军团数量：`Legions: 0`
- 舰队数量：`Fleets: 0`

#### 2.2 派系影响力（Faction Influence）
用颜色区分三个派系：
- **Optimates**（深红色）- 玩家所在派系，标注 `(YOU)`
- **Populares**（深绿色）
- **Equites**（深蓝色）

每个派系显示当前总影响力数值。

#### 2.3 派系人物列表（Faction Characters）
- 用 `*` 标记派系领袖
- 用 `O` 标记普通成员
- 显示人物姓名 + 关键属性（影响力数值）
- 支持点击展开/折叠详细信息

## 三、右侧操作面板（Right Panel）- 持久面板

### 3.1 操作按钮区（Action Buttons）

根据当前阶段动态显示可用的操作按钮：

- **通用按钮**（始终显示）：暂停、设置、帮助、查询派系金库
- **阶段专属按钮**：
  - 天命：行卜、跳过
  - 广场：查看人才、招募、竞标、购地、凯旋投票
  - 人口：查看候选人、竞选庆典、投票
  - 元老院：查询人物、提案、投票、否决
  - 战斗：侦查、扎营、进攻

按钮设计：
- 棕色背景 + 金色边框
- 悬停效果：边框加深 + 背景色变化
- 不可用时：灰色禁用状态

### 3.2 事件日志区（Event Log）

- 按时间倒序排列所有游戏事件
- 不同类型事件用颜色区分：
  - 警告（红色）：战争威胁、财政危机
  - 成功（绿色）：法案通过、战斗胜利
  信息（棕色）：日常通知、人物状态变化
- 支持滚动查看历史记录
- 点击事件可跳转到相关界面（如点击"新人才到达"跳转到人才市场）

## 六、交互设计要点

### 6.1 信息分层
- **一级信息**（始终可见）：国库、影响力、人物列表、当前阶段
- **二级信息**（点击展开）：人物详细属性、法案详情、战争详情
- **三级信息**（弹窗/新页面）：人物历史、派系历史、游戏设置

### 6.2 快捷键设计
- `Enter` / `Space`：确认/下一步
- `1-9`：快速选择操作按钮
- `Tab`：切换焦点区域
- `ESC`：打开暂停菜单
- `H`：打开帮助

### 6.3 动画效果（可选）
- 阶段切换：进度条平滑过渡
- 事件触发：事件卡片从上方滑入
- 投票结果：投票进度条动画填充
- 战斗结果：骰子滚动动画

## 七、技术实现建议

由于项目限制**仅使用 Python 标准库**，推荐以下技术方案：

### 方案 A：tkinter（标准库）
- **优点**：无需额外依赖，符合项目约束
- **缺点**：外观较为朴素，需较多自定义样式代码
- **适用**：简单风格完全符合用户要求

### 方案 B：HTML + 内置 HTTP 服务器（标准库 http.server）
- **优点**：美观灵活，可复用 CSS 样式
- **缺点**：需要浏览器支持，与 CLI 架构整合需额外工作
- **适用**：长期 GUI 发展路线

### 推荐实现路径
1. **第一阶段**：使用 tkinter 实现核心布局框架（左中右三栏 + 阶段切换）
2. **第二阶段**：逐个阶段实现主内容区（从广场阶段开始，因为交互最丰富）
3. **第三阶段**：添加动画效果、音效、主题切换
4. **第四阶段**：可考虑迁移到更现代的框架（如 PyQt 或网页）

## 八、与现有 CLI 架构的整合

当前架构规则：`UI → API → Systems → Entity → GameState`

GUI 实现应遵循此规则：
- **GUI 层**（tkinter）：负责渲染和捕获用户输入
- **API 层**（CommandRouter）：将 GUI 操作转换为现有命令格式
- **Systems 层**：复用现有逻辑，无需修改

这样 CLI 和 GUI 可以共存，通过不同前端调用同一套后端逻辑。
