# Eagle of Rome GUI 界面设计说明书 V2.0

> 更新说明：基于 OPC-01 方向确认（2026-07-05），以 `EOR_GUI_Prototype_v3.23.html` 为视觉参考基准。
> 本版本新增：完整视觉设计规范（§9）、全局查询操作栏（§4）、多玩家交互视觉模式（§10）。

## 设计概述

基于 Eagle of Rome 已开发功能（MVP 0.7）设计的 GUI 界面原型。采用**统一布局框架 + 阶段内容动态切换**的设计策略，确保玩家在不同游戏阶段中拥有稳定的导航心智模型。

## 设计原则

1. **三栏通顶通底**：7 阶段共享同一骨架，左=阶段导航，中=主内容，右=上下文+日志，顶=全局信息，底=全局查询
2. **操作即结果**：所有用户操作在主内容面板完成，结果在公示区和日志区展示
3. **面板锁定**：多玩家轮流操作时，非当前玩家面板自动锁定（半透明+模糊），当前玩家完成即解锁下一环节
4. **罗马风格配色**：羊皮纸背景 + 深红标题 + 古铜色边框，营造罗马共和国氛围

## 一、全局布局框架（三栏通顶通底）

```
+------------------------------------------------------------------+
|  [HEADER] 🏛️ EAGLE OF ROME · SPQR  |  国库 142  派系 12  影响力 68  |  48px
+------+-----------------------------------+------------------------+
|      |                                   |  [CONTEXT PANEL]      |
|  阶  |           CENTER PANEL            |  当前阶段状态总览       |
|  段  |      (阶段主内容区，含公示区        |  ──────────────       |
|  导  |        + 子环节面板)              |  状态条目1: xxx        |
|  航  |                                   |  状态条目2: xxx        |
|      |                                   |  ──────────────       |
|  7   |                                   |  [EVENT LOG]          |  240px
|  圆  |                                   |  > [10:32] 事件1       |
|  形  |                                   |  > [10:31] 事件2       |
|  按  |                                   |  > [10:30] 事件3       |

+------+-----------------------------------+------------------------+
|  [BOTTOM OP BAR] 📋派系 👤人物 📊状态 💰金库 🌾公地 🏡私地 …  |  40px
+------------------------------------------------------------------+
```

### 1. 顶部标题栏（Header Bar）— 48px 固定高度
- **左侧**：`🏛️ EAGLE OF ROME · SPQR`（Logo + 副标题 `em` 小字）
- **中部**：全局状态条（国库 💰、派系金库 👛、影响力 ⚖️、稳定度 🏛️、战争数 ⚔️），每项含数值+标签
- **右侧**：回合信息 `282 BC · 回合 1`
- **视觉**：`--header-bg: #8B2500` 深红色背景 + `--header-text: #FFD700` 金色文字
- 状态项之间用 `border-left: 1px solid rgba(255,255,255,0.15)` 分隔

### 2. 左侧阶段导航栏（Phase Rail）— 44px 固定宽度

7 个圆形按钮竖排，每个 34×34px：

| 序号 | 图标 | 阶段 | 显示状态 |
|------|------|------|----------|
| 1 | 🎴 | 天命 | 已完成(`done`)=`#CD853F` 白字 |
| 2 | 💰 | 收入 | 当前(`current`)=`#E8B84B` 金色+发光 |
| 3 | 🏛️ | 广场 | 未到(`todo`)=`#E8D5C4` 灰字 |
| 4 | ⚖️ | 人口 | — |
| 5 | 🏺 | 元老院 | — |
| 6 | ⚔️ | 战斗 | — |
| 7 | 📊 | 决算 | — |

- 当前阶段：金色背景 + 古铜色边框 + `box-shadow` 发光
- 已完成阶段：秘鲁色 `#CD853F`
- 未到阶段：浅灰色 `#E8D5C4`
- 悬停：`scale(1.1)` 放大 + 弹出标签（`span.rl`）显示阶段中文名
- 点击可回看已完成阶段的内容

### 3. 中间主内容区（Center Panel）— flex:1 自适应

- 背景 `--panel-bg: #FAF5EE` + 古铜色边框
- `padding: 14px`，纵向弹性布局
- 阶段内容切换：CSS class `active` 控制 display:none/display:flex
- 切换动画：`fadeIn` 0.2s ease（opacity 0→1 + translateY 4px→0）

### 4. 右侧上下文面板（Context Panel）— 240px 固定宽度

上部分（Context Top）：
- 当前阶段操作按钮（`ctx-btn` 样式，全幅，左对齐）
- 当前阶段状态总览（`stat-tile` 样式，条目式 key:value）
- 刷新/重置按钮

下部分（Event Log）：
- 深色终端风格日志区，时间倒序
- 每条日志格式：`[时间戳] 事件内容`
- 颜色分类：
  - `fb-success`（绿色）= 法案通过、战斗胜利
  - `fb-warning`（橙色）= 风险、提醒
  - `fb-error`（红色）= 失败、危机
  - `fb-info`（棕色）= 日常通知
- 可滚动，自定义滚动条 4px 宽

### 5. 底部操作栏（Operation Bar）— 40px 固定高度

深红色 `#6B1C00` 背景，水平排列 12 个全局查询按钮：

| 按钮 | 查询内容 |
|------|----------|
| 📋 派系信息 | 各派系国库/影响力/成员数 |
| 👤 人物查询 | 输入ID查看详细属性 |
| 📊 游戏状态 | 国库/公地/存活人物/军团等 |
| 💰 派系金库 | 各派系金库余额 |
| 🌾 公地信息 | 国家公地/已分配/待售/行省公地 |
| 🏡 私地信息 | 查看私有土地持有 |
| 📦 合同状态 | 待决/执行中/已完成合同 |
| 🏛️ 行省信息 | 各总督/空缺行省 |
| ⚔️ 战争列表 | 活跃/威胁/停战状态 |
| 🗡️ 军团状态 | 各军团位置/编制 |
| ⚓ 舰队状态 | 各舰队位置/战力 |
| ❓ 帮助 | 游戏帮助信息 |

- 按钮样式：半透明白底 `rgba(255,255,255,0.08)` + 浅边框 + 白色文字
- 悬停：古铜色高亮 `rgba(212,175,55,0.25)`
- 点击查询结果以模态弹窗展示

- 三栏 `flex`：我方 `wc-ally` 绿色 / VS / 敌方 `wc-enemy` 红色
- 数值 `1.1rem` 加粗，标签 `0.58rem` 浅色
- 多战争并排显示（最多 3 个）

### 6.9 法案配置面板（.bill-config）
- 可折叠配置区，展开/收起箭头动画
- 下拉选择 `bc-select`：`width:100%`，圆角 `4px`
- 滑杆 `bc-range`：`height:4px`，`accent-color:--accent`
- 滑杆值实时显示，`accent` 色加粗
- 自动接手法案标记 `bc-badge-auto`（警告色）

### 6.10 弹性布局行（Flex Row）
- `cand-row` 候选人行：`flex` 分布，各列宽度分配
- `fest-row` 庆典行：同 flex 分配
- `talent-row` 人才行：同 flex 分配，含阶级/属性/费用

## 七、7 阶段子环节与交互概览

### 7.1 天命阶段（Mortality）— 1 子环节
- **子环节 1**：点击「⚡ 执行天命」按钮触发随机事件
- 按钮居中，执行后灰化
- 事件结果在公示区卡片展示
- 阶段描述：`众神降下命运——触发一个随机事件。无玩家操作。`
- 多玩家：任意一名玩家执行一次

### 7.2 收入阶段（Revenue）— 1 子环节
- **子环节 1**：查看收支明细（国家收入/支出 + 派系财政 + 地主收益）
- 国家收入/支出左右并排
- 派系财政：国库拨款 + 会员贡献，合计显示
- 地主收益：可点击人物查看派系和收入（不显示财富）
- 底部金色框显示净变化和新余额
- 点击「确认收入结算」完成

### 7.3 广场阶段（Forum）— 2 子环节
- **布局**：公示区（顶部）+ 2 个子环节面板并排（`.forum-panels`）
- **子环节 1**：解雇成员（本派系成员勾选→解雇→自动进入人才市场）
- **子环节 2**（锁定）：招募/竞标/公地认购/凯旋投票
- 子环节 2 初始锁定，子环节 1 完成后自动解锁
- 面板标题样式：`--header-bg` 深红色 + 金色文字 + 序号圆标
- 所有玩家完成 → 公示区展示结果

### 7.4 人口阶段（Population）— 2 子环节
- **子环节 1**：庆典赞助（为本派系候选人提供赞助，费用来自个人财富）
- **子环节 2**（锁定）：投票选举（每官职选一名候选人，可不选）
- 公示区表格：6 列（官职/候选人/四属性/影响力/派系/选举结果）
- 属性拆为 4 独立列：⚔️ 军略/🧠 智略/💬 魅力/🔥 热忱

### 7.5 元老院阶段（Senate）— 3 子环节
- **子环节 1**：执政官提案（8 种法案类型，各有不同参数配置面板）
  - 宣战/停战/总督任命/建造合同/包税合同/卖地/分地/接管战争
  - 接管战争绕过元老院表决
- **子环节 2**（锁定）：元老表决（勾选同意，未勾选=否决）
- **子环节 3**（锁定）：保民官否决（勾选否决，未勾选=同意）
- **死锁逃生**：无法案通过→跳过否决→完成按钮；全否决→否决面板内追加完成按钮

### 7.6 战争阶段（Combat）— 1 子环节
- 多战争多面板并排（最多 3 场，第 4 场游戏结束）
- 每场战争含：军力对比卡片 + 指挥官/军团信息 + 进攻按钮
- 进攻按钮每回合一次，点后灰化
- 平局→自动生成和约草案→下回合元老院表决
- 预览区显示共和国军力总览（不展示战斗结果）

### 7.7 决算阶段（Resolution）— 0 子环节
- 无操作交互
- 总督轮换自动计算
- 显示年度总结

## 八、模态弹窗（Modal Dialog）

- **遮罩层**：固定定位，全屏覆盖，`rgba(0,0,0,0.35)`
- **弹窗容器**：居中，`--panel-bg` 背景，`--bronze` 2px 边框
- **标题**：`--header-bg` 红色，`border-bottom:2px solid --panel-border`
- **内容行**：`modal-row` 样式，label:value 左右分布，虚线分隔
- **关闭按钮**：右上角 `×`，`1.2rem`
- **按钮行**：`modal-btn-row` flex 靠右排列「确认」「关闭」

## 九、多玩家交互视觉模式

### 9.1 玩家轮次切换
- 当前玩家在 Header 中高亮显示
- `player_api.is_current_player()` 控制交互锁定
- 非当前玩家面板自动进入 `.panel-locked` 状态
- AI 玩家使用 `Auto*Decider` 后台执行，不占用 UI 轮次

### 9.2 各阶段多玩家规则
| 阶段 | 规则 |
|------|------|
| 天命/收入/决算 | 任意一名玩家执行一次即可 |
| 广场/人口 | 子环节内轮流 → AI 后台 → 解锁下一步 |
| 元老院 | 执政官提案（执政官玩家）→ 所有玩家表决 → 保民官否决 |
| 战争 | 任意玩家分别进攻各战争（按钮灰化独立追踪） |

### 9.3 玩家交接动画
- 当前玩家完成 → 面板淡出 → 下一玩家面板淡入（`fadeIn` 0.2s）
- 交接时显示 `PlayerHandoffOverlay` 提示当前轮到谁
