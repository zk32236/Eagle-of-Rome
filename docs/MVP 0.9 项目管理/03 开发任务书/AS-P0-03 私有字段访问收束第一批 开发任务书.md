## 任务目标

在不改变现有玩法和数值规则的前提下，收束第一批关键路径私有字段访问：

1. 人口阶段 pending 统一由 `GameState` 公共接口管理。
2. 自动庆典和自动投票统一调用 `population_api`，不再直接修改人物和 pending。
3. 广场、人口和海军路径不再直接读取或修改 `WarSystem` 私有容器。
4. 年末总督交接通过 `Province` 原子公共方法完成。
5. 公地认购通过 `Figure` 公共土地方法完成人物财富与私地变更。

本任务不得扩大为全项目私有字段清理。

## 实现要求

### 一、人口 pending 公共接口

在 `GameState` 增加业务语义明确的方法，命名可按项目风格微调，但必须覆盖：

- 记录庆典。
- 返回庆典记录副本。
- 查询、记录投票；支持自动玩家覆盖同一玩家同一官职旧票。
- 返回投票记录副本。
- 返回人口 pending 快照。
- 清空人口 pending。

要求：

- 返回值不得允许调用方修改内部列表。
- 投票唯一性规则由公共方法或 API 单点维护。
- `population_api.campaign()` 和 `vote()` 增加显式 `bypass_permission: bool = False`。
- 默认路径保持当前玩家和派系权限检查。
- bypass 只跳过玩家回合/派系权限，不得跳过人物存在、候选资格、金额、财富等业务校验。
- `resolve_election()` 只通过公共查询读取投票。
- `phase_population.py` 只通过公共查询统计庆典支出，并通过公共方法清理。

### 二、自动模式统一 API

修改 `AutoPlayerProcessor.process_festival()` 和 `process_vote()`：

- 无论是否 bypass，都调用 `population_api.campaign()` / `vote()`。
- `bypass_permission=True` 时仅通过 API 显式参数绕过权限。
- 删除自动模式直接扣财富、加人气、改影响力、追加或替换 pending 的代码。
- 保持现有日志语义，避免重复记录相同事件。

### 三、WarSystem 公共接口

增加或补强以下受控入口：

- 查询已解决/弃牌区战争，返回副本。
- 查询需要海战的威胁战争，返回副本。
- 登记由 `create_rebellion_war()` 创建的活跃起义战争，防止重复登记并确保状态为 ACTIVE。
- 复用现有 `clear_legions_to_disband()`。

迁移调用：

- `forum_api.py`、`phase_forum.py`、`auto_player_processor.py`、`phase_population.py` 不再读取 `_war_discard`。
- `naval_system.py` 不再读取 `_threats`。
- `phase_forum.py` 不再向 `_active_wars` 追加起义战争。
- `phase_population.py` 不再读取或清空 `_legions_to_disband`。

军团解散要求：

- 从 `clear_legions_to_disband()` 取得待处理编号。
- 如解散函数返回失败编号或错误，必须将未成功处理的编号重新登记，不能静默丢失。
- 保持当前输出和日志。

### 四、总督交接

在 `Province` 增加一个原子交接方法，例如：

`complete_governor_transition(turn: int) -> tuple[Optional[int], Optional[int]]`

语义要求：

- 返回本轮旧总督 ID 与候任总督 ID。
- 有候任人时，将其设为当前总督并更新任期起始回合。
- 无论候任人是否有效，完成调用后均清理候任和旧总督临时字段。
- 不负责修改 Figure；人物返回罗马、上任、office、is_absent、影响力、显示和日志仍由 `phase_resolution.py` 编排。

`phase_resolution.py` 不得再直接读取或修改 `_old_governor_id`、`_governor_designate_id`、`_governor_id`。

### 五、公地认购

`forum_api.resolve_forum()` 的公地认购结算：

- 使用 `Figure.buy_land(actual_buy, land_price)` 完成人物扣款和私地增加。
- API 继续处理国库增加、国家公地减少、配额减少和结果信息。
- 调用失败不得继续修改国库或公地。
- 不得重复扣人物财富。
- 保持影响力更新、影响力排序、资金上限和配额规则。
