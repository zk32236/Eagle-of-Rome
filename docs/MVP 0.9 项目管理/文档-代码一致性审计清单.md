## 2. 核心实体一致性

| 实体 | 文档预期 | 代码检查点 | 结果 |
| --- | --- | --- | --- |
| `GameState` | 中心状态容器 | 注册表、玩家、阶段、临时数据、序列化 |  |
| `Province` | 位于 `province.py` | 不应误用 `entities.py` 中旧类 |  |
| `Contract` | 合同生命周期完整 | 状态、竞标、中标、完成、到期 |  |

| `Fleet` | 舰队建造、战力、指派 | 合同完成同步 |  |
| `Legion` | 军团征召、维护、恢复 | 状态流转正确 |  |
| `Player` | 多玩家顺序与权限 | 当前玩家和派系绑定 |  |
| `Curia` | 广场等待区 | 招募和清理流程正确 |  |
| `City` | 预留城市系统 | 不影响当前流程 |  |

## 3. API 一致性

| 检查项 | 预期 | 风险点 |
| --- | --- | --- |
| API 返回格式 | `{success, message, data, errors}` | 不得重复定义或返回旧格式 |
| `forum_api` | 退休、招募、竞标、买地、凯旋、土地交易、结算 | 土地交易不得绕过 API |
| `senate_api` | 初始信息、提案、投票、否决、结算、战争接管 | 不得承担过多政治核心规则 |
| `population_api` | 庆典、投票、候选人、选举结算 | 临时数据访问需收束 |
| `game_api` | 阶段执行、回合推进、状态查询 | 避免 API 反向导入 UI 命令 |
| 查询 API | 人物、派系、行省、合同、玩家 | 输出结构稳定 |

## 4. 分层依赖一致性

| 检查项 | 允许 | 不允许 | 结果 |
| --- | --- | --- | --- |
| UI 命令层 | 调用 API、格式化输出 | 直接改核心私有字段 |  |
| API 层 | 调用系统/服务/GameState 公共方法 | 导入 UI 命令类 |  |
| 系统/服务层 | 调用实体和 GameState 公共方法 | 依赖 UI 输入输出 |  |
| 实体层 | 自身状态和局部行为 | 依赖 API/UI |  |
| 决策器 | 返回决策结果 | 直接改业务状态 |  |

| 位置 | 是否允许 | 处理建议 |
| --- | --- | --- |
| 实体内部 | 通常允许 | 保持封装 |
| GameState 内部 | 通常允许 | 优先提供公共方法 |
| 系统/服务层 | 谨慎允许 | 尽量通过公共方法 |
| API 层 | 不推荐 | 新增 GameState/系统方法 |
| UI 命令层 | 不允许 | 迁移到 API/系统/服务 |
| 测试代码 | 可有限允许 | 但不应作为业务模式 |
