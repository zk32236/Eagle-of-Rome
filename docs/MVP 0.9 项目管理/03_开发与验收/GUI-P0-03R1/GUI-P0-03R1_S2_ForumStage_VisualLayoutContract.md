## 4. Layout Contract

### 4.1 主内容结构

| 区域 | 契约 |
| --- | --- |
| 容器 | ForumStage 只能填充 StageContentSlot，不重写 Shell |
| 内容宽度 | 左右双卡片在中央象牙白桌面内，左右间距稳定 |
| 卡片数 | Phase 3A/3B 均保持双卡片：左解雇、右市场 |
| 卡片标题 | 固定高度，不被内容挤压 |
| 溢出策略 | 内容超出卡片高度必须滚动，不得覆盖底部按钮 |

### 4.2 左侧解雇成员卡片

| 项目 | 契约 |
| --- | --- |
| 列表区域 | 独立 ScrollView / ListView，占据卡片标题与底部按钮之间空间 |
| 行高 | 固定或最小高度，建议 `>= 44px`；不得依赖自然内容高度导致重叠 |
| 姓名列 | 占主要宽度，超长姓名使用省略或固定宽度，不得压到按钮 |
| 属性列 | 固定宽度显示身份/影响力，不得与姓名重叠 |
| 按钮列 | 固定宽度，位于行右侧；按钮不得覆盖姓名或属性 |
| 锁定态 | 锁定按钮仍占同一按钮列宽度，避免布局跳动 |
| 底部按钮 | 固定在卡片底部，满宽或接近满宽；不随列表滚动；不得覆盖列表行 |

### 4.3 右侧市场卡片

| 项目 | 契约 |
| --- | --- |
| 内容区域 | 独立滚动，保留人才、合同、公地、凯旋四块 |
| 人才表 | 列宽固定：姓名、年龄、智力、魅力、热忱、阶级、费用、按钮 |
| 合同区 | PENDING/BUDGETED 状态必须清晰，PENDING 不可竞标 |
| 公地区 | 无额度时显示空状态，保留栏位 |
| 凯旋区 | 无可表决战争时显示空状态，保留栏位 |
| 底部按钮 | 市场提交/完成下注按钮位于市场内容底部，不能替代 ContextPanel 推进按钮 |

### 4.4 禁止重叠规则

以下任一出现即直接退回：

- 人物姓名与按钮重叠。
- 人物属性与姓名重叠。
- 按钮覆盖列表文本。
- 底部按钮覆盖列表行。
- 卡片标题被内容挤压。
- 超出高度但没有滚动行为。
