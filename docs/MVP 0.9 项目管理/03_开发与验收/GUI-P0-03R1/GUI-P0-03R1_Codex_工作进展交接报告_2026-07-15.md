## 4. 市场环节已确认的产品决策

以下是 PM 已确认的规则，后续 OC/SA/DA 必须遵守：

1. 市场环节使用真实游戏数据，不使用视觉校准假数据。
2. 人才市场人物不应在解雇前提前公开；如果 CORE 当前逻辑是在市场阶段生成新人，则 GUI 应尊重该逻辑。
3. 第一回合可能没有正式招标合同，但应显示 Pending Contract / 待元老院预算表决合同信息。
4. 第一回合可能没有公地认购，但区块位置可以保留。
5. 市场阶段右侧市场区底部应提供“提交下注”按钮。
6. 当玩家可以配置招募、竞标、认购、凯旋投票时，按钮显示“提交下注”。
7. 所有玩家下注完成后，按钮变为“完成下注”且不可操作。
8. 全部玩家完成后，才允许通过右侧 ContextPanel 的“推进到下一阶段”进入人口阶段。
9. 步骤条只保留两个编号：
   - `1 解雇成员`
   - `2 市场（招募·竞标·认购·凯旋）`
10. 市场环节标题应写为：`市场（招募·竞标·认购·凯旋）`。
