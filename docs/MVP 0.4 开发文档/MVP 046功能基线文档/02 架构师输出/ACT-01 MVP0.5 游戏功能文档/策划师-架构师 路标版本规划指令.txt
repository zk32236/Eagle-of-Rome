《罗马鹰旗》EOR MVP0.5-MVP1.0 版本路标重构（按「可玩特性」拆分里程碑）
核心诉求
基于现有 MVP0.5 架构设计成果，重构 MVP0.5 至 MVP1.0（最小可玩版）全流程开发里程碑，核心要求为每个里程碑仅聚焦 1-2 个核心可玩特性，且每个里程碑开发完成后，能实现独立、可直接试玩、可直观验证的游戏功能输出，让开发过程有明确反馈、能积累成就感，同时严格遵循「增量开发、不推翻现有架构、贴合罗马共和国核心玩法」原则，最终落地 3-4 人可玩的最小完整版本。
前置基础
已完成 MVP0.5 架构设计，输出 5 份核心文档（命令系统 / 开发约束 / 架构总览 / 标准化接口 / 模块调用与业务流程），落地「土地→金钱」核心循环为基础；
目标终态为 MVP1.0：支持 3-4 人博弈、核心玩法闭环（土地 / 金钱 / 影响力联动）、基础胜负判定、CLI 可流畅试玩的最小可玩版；
所有里程碑需基于现有架构做增量开发，无架构重构，下游开发可直接基于里程碑要求落地，无需额外确认设计。
重构要求
1. 里程碑拆分原则
数量控制：拆分4-5 个里程碑（含 MVP0.5 落地版），避免过细 / 过粗，单个里程碑开发完成后 1-2 小时可完成试玩验证；
特性聚焦：每个里程碑仅围绕1 个核心可玩特性 + 少量配套功能，不叠加无关逻辑，核心特性需贴合「罗马共和国政治经济博弈」核心玩法；
可玩闭环：单个里程碑的核心特性需实现「操作 - 反馈 - 结果」完整闭环，比如「竞标合同」需实现「生成合同→玩家出价→中标结算→财富变化」全流程，可独立试玩；
渐进迭代：里程碑按「从基础经济到政治博弈、从单人试玩到多人对抗」递进，后一里程碑基于前一里程碑增量，逐步完善核心玩法，最终汇聚为 MVP1.0 完整版本；
验证明确：每个里程碑需明确试玩验证标准（含操作步骤、预期反馈、可验证结果），开发完成后可按标准快速验证，无模糊点。
2. 里程碑内容要求
每个里程碑需明确包含6 大核心要素，结构统一、信息无歧义，直接作为开发 / 验收依据：
里程碑编号 + 名称（如 M1：MVP0.5 土地与基础合同可玩版）；
核心可玩特性（1 个，明确核心玩法点）；
增量开发内容（基于现有架构，仅列新增 / 完善功能，无冗余）；
试玩验证标准（步骤化、可落地，含操作指令、预期游戏反馈、可观测结果）；
架构适配要求（明确与现有 MVP0.5 架构的衔接点，无架构修改）；
与下一级里程碑的衔接（明确本里程碑为后续提供的功能 / 数据基础）。
3. 核心可玩特性方向（需覆盖且仅覆盖以下范围，不新增额外玩法）
基础经济闭环：土地收支、合同竞标（包税权 / 公共工程）的单人可玩；
派系基础操作：派系土地管理、骑士合同归属的派系联动；
多人基础对抗：3-4 人权限隔离、回合轮转、多人合同竞标博弈；
影响力基础联动：金钱向影响力的转化（如庆典、土地持有量关联影响力）；
最小胜负闭环：基础胜利 / 失败判定、游戏终止条件、结算排名。
输出要求
输出格式：结构化 Markdown，分里程碑逐一列明，每个里程碑 6 大核心要素清晰分隔，便于阅读和落地；
语言要求：技术简洁、无模糊表述，试玩验证标准需CLI 指令化（可直接输入游戏执行），预期结果需可量化 / 可观测（如财富数值变化、合同状态变更、派系信息更新）；
落地性：所有内容基于现有 MVP0.5 架构设计，无新增架构设计要求，开发人员可直接按里程碑开展增量开发；
终态对齐：最后一个里程碑直接落地 MVP1.0 最小可玩版，满足 3-4 人可玩、核心玩法闭环、CLI 流畅试玩的核心要求。
参考示例（里程碑结构）
里程碑 M1：MVP0.5 土地收支 + 单人合同竞标可玩版
核心可玩特性：单人模式下，完成「土地收支自动结算 + 包税权合同单人竞标」全流程经济操作，实现基础金钱收益与消耗的可玩闭环；
增量开发内容：①落地 Province/Contract 实体核心代码；②实现税收阶段土地收支自动结算；③实现广场阶段包税权合同生成 + 单人骑士出价竞标；④完善对应 CLI 查询指令（status_contract/status_province）；
试玩验证标准：①输入revenue触发税收阶段，可观测「私地收入计入骑士财富、公地租金计入国库」（数值可查）；②输入forum触发广场阶段，自动生成包税权合同；③输入bid_contract 1 100完成竞标，可观测「骑士财富扣减、国库增加、合同标记中标、骑士绑定合同」（全状态可查）；
架构适配要求：严格遵循现有 MVP0.5 实体定义 / 接口设计 / 命令框架，无架构修改；
与下一级里程碑衔接：为 M2 公共工程合同玩法提供「合同竞标基础流程 + 金钱收支体系」基础。
请基于以上全部要求，完成《罗马鹰旗》EOR MVP0.5-MVP1.0 可玩特性导向的开发里程碑重构，输出完整的版本路标规划。
