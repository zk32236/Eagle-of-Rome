核心任务：

创建军事功能命令文件：

路径：src/ui/commands/func_military.py

该文件应包含五个命令类：WarsCommand、AssignCommand、LegionsCommand、RecruitCommand、DisbandCommand，均继承 Command。

WarsCommand：

name = "wars"

移植业务逻辑：

WarsCommand：

从原 _show_wars 方法移植逻辑。

使用 self.state.get_war_system() 获取战争系统，若不存在则打印信息并返回 False。

调用战争系统的相关方法（如 get_active_wars、_war_deck、_war_discard 等）获取数据，并按照原格式打印输出。

注意显示指挥官伤亡状态、军团数量等细节。

AssignCommand：

从原 _assign_commander_interactive 方法移植逻辑。

该方法包含多步交互：选择战争、选择指挥官、选择军团、指派海军等。

使用 input() 获取用户输入，需要处理输入转换和验证。

调用 war_system.assign_commander() 和 military_system.assign_to_war() 执行指派。

注意处理指挥官已存在时的替换/增援选项。

打印适当的成功/失败信息。

LegionsCommand：

从原 _show_legions 方法移植逻辑。

使用 self.state.get_military_system() 获取军事系统，调用其 get_military_summary() 和 display_legion_status() 输出军团状态。

RecruitCommand：

从原 _recruit_legion_interactive 方法移植逻辑。

支持单军团和批量征召两种模式（通过用户选择）。

检查国库资金，调用 military_system.recruit_legion() 或 recruit_multiple()。

输出征召结果。

DisbandCommand：

从原 _disband_legion_interactive 方法移植逻辑。

列出未指派的军团，支持批量输入（逗号分隔或 all）。

调用 military_system.disband_legion() 执行解散。

输出解散结果。

辅助方法：

可在文件内定义辅助函数，用于处理交互式输入、显示列表、输入验证等，避免重复代码。

对于需要多次获取的战争系统、军事系统，可在每个命令的 execute 中获取并检查是否为 None。

编写单元测试：

测试用例至少覆盖：

AssignCommand：

测试指挥官已存在时的替换/增援选项。

测试用户取消操作。

LegionsCommand：

验证输出包含军团摘要和状态。

RecruitCommand：

测试单军团征召（成功/资金不足）。

测试批量征召。

测试用户取消。

DisbandCommand：

测试单军团解散、批量解散、无效输入。

注意模拟 war_system 和 military_system 的方法，避免真实操作。

执行 disband 解散部分军团。

确保所有命令无崩溃，输出合理。
