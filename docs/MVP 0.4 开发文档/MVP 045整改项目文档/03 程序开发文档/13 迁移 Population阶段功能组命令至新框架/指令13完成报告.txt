核心任务：

创建 Population 功能命令文件：

路径：src/ui/commands/func_population.py

该文件应包含一个命令类：FestivalCommand，继承 Command。

元数据定义：

name = "festival"

aliases = []（原基线无别名，可保持）

description = "举办庆典，消耗个人财富增加人气，用法: festival <人物ID> <金额>"

移植业务逻辑：

从原 _hold_festival 方法移植逻辑。

命令接收两个参数 args，应为 [figure_id, amount]。需验证参数个数和类型。

将 figure_id 和 amount 转换为整数，若转换失败则打印错误并返回 False。

使用 self.state.get_member(figure_id) 获取人物对象。若人物不存在或已死亡，打印错误并返回 False。

获取玩家派系：遍历 self.state.factions.values()，找到 is_player == True 的派系。若不存在，打印错误并返回 False。

验证人物是否属于玩家派系：检查 figure.faction_id 是否等于玩家派系 ID。若不属于，打印错误并返回 False。

验证人物财富是否足够：figure.wealth >= amount。若不足，打印错误并返回 False。

扣除财富：figure.wealth -= amount。

增加人气：原基线中人气增加量等于花费金额（1:1）。可调用 figure.add_popularity(amount) 或直接 figure.popularity += amount。建议使用 add_popularity 方法（如果存在）。

打印成功信息，包含人物名称、花费金额、增加的人气、当前人气和财富。

注意：原基线输出格式为：

text
🎉 {figure.get_formal_name()} holds a festival!
   Spent {amount} personal funds
   Gained {amount} popularity
   Current: {figure.popularity} popularity, {figure.wealth} wealth
需保持一致。

辅助方法：

可在命令内部实现 _get_player_faction 辅助方法，与 PersuadeCommand 类似。

整数转换使用 try/except。

test_festival_invalid_args：参数个数错误、非数字参数，返回 False。

test_festival_insufficient_wealth：人物财富不足，返回 False。

注意模拟 state.get_member 的返回值，以及人物属性的修改。
