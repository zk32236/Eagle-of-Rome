移植业务逻辑：

从原 senate_phase.py 的 SenatePhase.execute() 方法复制核心逻辑，适配到新命令的 execute(self, args: List[str]) -> bool 中。

关键适配点：

使用 self.state.get_living_members() 获取存活人物列表。

使用 self.state.get_faction() 获取派系。

使用 self.state.get_offices_per_election(office_type) 获取各官职选举人数。

使用 self.state.get_office_cooldown(office_type) 获取冷却期。

使用 self.state.get_presiding_officer() 获取主持人。

使用 self.state.turn.leader_ids 获取当前执政官列表（若 GameState 中 _turn 尚未实现，需先补充 leader_ids 属性或通过公共方法访问）。

使用 self.state.contracts 访问合同列表，并调用合同的相关方法（如 award）。

使用 self.state.log_event() 记录事件。

使用 self.state.is_phase_executed("senate") 检查阶段是否已执行，若已执行则打印提示并返回 False。

执行成功后调用 self.state.mark_phase_executed("senate")。

原代码中的多个私有方法（如 _elect_single_magistrate、_process_pending_contracts、_vote_tax_contract、_assign_works_contract、_show_seat_standings 等）应作为命令的私有方法移植，保持逻辑完整。

注意处理选举逻辑中的人物资格检查（can_hold_office 方法已在 Figure 类中存在，可直接调用）。

合同授予部分需调用 contract.award() 方法，并更新相关人物财富和国库（使用 self.state.add_figure_wealth 和 self.state.add_treasury）。

确保所有打印输出与原基线一致（保留原有的输出格式和 emoji）。
