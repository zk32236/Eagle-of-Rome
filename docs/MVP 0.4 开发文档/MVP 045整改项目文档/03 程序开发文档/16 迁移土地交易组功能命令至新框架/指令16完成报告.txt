创建土地交易功能命令文件：

路径：src/ui/commands/func_land.py

该文件应包含三个命令类：TradeLandCommand、LandPriceCommand、SeatsCommand，均继承 Command。

元数据定义：

TradeLandCommand：

name = "trade_land"（注意原命令为 trade land，但命令名不能包含空格，因此用下划线连接，用户输入仍需为 trade land <seller> <buyer> <amount>。在命令解析时，第一个词是 trade，第二个词是 land 作为参数。因此我们需要一个 TradeCommand 类，然后在 execute 中判断 args[0] == "land"。类似 vote 命令的处理。为了保持一致，这里采用与 vote 相同的策略：创建 TradeCommand，命令名 "trade"，然后在 execute 中解析子命令。

aliases = []

description = "执行土地交易，用法: trade land <卖家ID> <买家ID> <数量>"

LandPriceCommand：

命令名 "land_price"（原命令 land price，类似处理：创建 LandCommand，命令名 "land"，在 execute 中判断 args[0] == "price"。但为了简化，也可单独作为 LandPriceCommand，命令名 "land_price"，但用户输入需为 land_price 不符合习惯。故采用 LandCommand 方式，命令名 "land"，子命令 price。

aliases = []

description = "预览土地交易价格，用法: land price <卖家ID> <买家ID>"

SeatsCommand：

name = "seats"

aliases = []

description = "显示席位占比分布（全国资产、个人席位、派系席位）"

移植业务逻辑：

TradeCommand（处理 trade land）：

命令名 "trade"，在 execute(self, args) 中首先检查 args 长度至少为 4，且 args[0].lower() == "land"，否则打印用法。

解析 args[1] 为卖家ID，args[2] 为买家ID，args[3] 为数量，均为整数。若转换失败则报错。

创建 LandTradingService 实例：service = LandTradingService(self.state)。

调用 service.execute_trade(seller_id, buyer_id, amount)，获取 (success, msg)。

根据 success 打印 msg，并返回 success。

LandCommand（处理 land price）：

命令名 "land"，在 execute(self, args) 中检查 args 长度至少为 3，且 args[0].lower() == "price"，否则打印用法。

解析 args[1] 为卖家ID，args[2] 为买家ID，为整数。

创建 LandTradingService 实例，调用 service.get_trade_preview(seller_id, buyer_id, 1) 获取预览信息（预览1单位价格）。

若预览信息为 None，打印错误；否则按原格式输出价格信息。

SeatsCommand：

从原 _show_seat_standings 方法移植逻辑。

使用 self.state.get_living_members() 获取所有存活人物。

计算全国总资产（土地+私兵），按比例计算个人席位和派系席位。

按照原格式打印排名和派系总计。

注意处理 total_assets == 0 的情况。

辅助方法：

可在文件内定义通用的整数解析辅助函数（如 _parse_int）以避免重复代码。

土地交易服务需要导入 from src.core.service.land_trading_service import LandTradingService。
