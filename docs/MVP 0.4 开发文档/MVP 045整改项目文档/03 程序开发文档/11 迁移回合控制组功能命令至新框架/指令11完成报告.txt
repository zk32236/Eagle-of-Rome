核心任务：

创建回合控制命令文件：

路径：src/ui/commands/func_turn_control.py

该文件应包含三个命令类：NextCommand、TurnCommand、StepCommand，均继承 Command。

元数据定义：

NextCommand：

name = "next"

aliases = ["n"]（原基线支持 n 作为别名）

description = "推进到下一年。用法: next [force] (force 可跳过未完成阶段检查)"

TurnCommand：

name = "turn"

aliases = []（原基线无别名）

description = "自动执行完整回合（依次执行所有7个阶段）"

StepCommand：

name = "step"

aliases = []

description = "逐步执行回合，每阶段后暂停等待输入"

移植业务逻辑：

NextCommand：

从原 _next_turn 方法移植逻辑。

检查 args 是否包含 "force" 字符串（不区分大小写），决定是否强制推进。

调用 self.state.is_phase_executed() 检查所有阶段是否已执行（可硬编码阶段列表或从 self.state 获取，但原基线有 phase_sequence 列表，可考虑在命令内部定义）。

若有缺失且未 force，打印缺失阶段列表，返回 False。

若缺失但 force，打印警告并继续。

调用 self.state.advance_year() 推进回合，打印新年份。

调用 self.state.get_status_summary() 或自定义输出显示回合摘要（原基线会显示阶段进度，可参考移植）。

TurnCommand：

从原 _do_full_turn 方法移植逻辑（auto=True 版本）。

获取当前回合已执行阶段，按顺序（mortality → revenue → forum → population → senate → combat → resolution）依次执行每个未执行的阶段。

对每个阶段，获取对应的命令类并执行（可通过注册器或直接实例化阶段命令类）。建议使用注册器执行，保持统一：self.state.registry.execute(phase_name, self.state, [])，但注意 TurnCommand 实例没有注册器引用，需要从外部传入或通过全局方式获取。一个可行方案是在命令中通过 self.state 获取注册器？但 GameState 并不持有注册器。因此更简单的方式是直接实例化阶段命令类（如 MortalityCommand(self.state).execute([])），但需导入所有阶段命令类。考虑到可维护性，可以在命令文件中导入所有阶段命令类并依次调用。

执行每个阶段后，不暂停（auto=True）。

执行完成后打印回合完成信息。

StepCommand：

从原 _do_full_turn 方法移植逻辑（auto=False 版本）。

与 TurnCommand 类似，但每执行一个阶段后，使用 input(" Press Enter...") 暂停等待用户按键。

注意处理 KeyboardInterrupt 等情况，确保不崩溃。

辅助方法：

可在文件内定义私有辅助方法 _execute_phase(phase_name) 来执行单个阶段，避免重复代码。

阶段顺序可定义为类常量或模块级列表：PHASE_SEQUENCE = ["mortality", "revenue", "forum", "population", "senate", "combat", "resolution"]。
