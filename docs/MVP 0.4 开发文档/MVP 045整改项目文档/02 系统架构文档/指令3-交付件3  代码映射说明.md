# 代码映射说明

本文档将架构设计中的核心元素映射到具体的代码文件、类和方法，确保设计与实现一一对应。

## 1. 配置类定义
- **文件**：`src/core/config.py`
- **类名**：`Config`
- **说明**：独立的配置管理类，与 GameState 解耦。

## 2. 内置默认配置
- **属性**：`Config.DEFAULTS`
- **类型**：`dict`
- **说明**：硬编码的默认配置字典，包含所有配置项的默认值。

## 3. 配置文件加载
- **方法**：`Config._load_from_file()`
- **职责**：读取 `data/config/game_config.json`，解析 JSON，与当前配置合并。
- **异常处理**：捕获 `FileNotFoundError`, `json.JSONDecodeError`, `PermissionError`, `TypeError`，并打印警告。

## 4. 首次初始化
- **方法**：`Config.__init__(config_path=None)`
- **职责**：调用 `_load_from_file()`，若失败则使用 `DEFAULTS`。

## 5. 运行时重载
- **方法**：`Config.reload()`
- **职责**：重新调用 `_load_from_file()`，若成功则更新 `self._config`，否则保持原配置。

## 6. 点号路径获取
- **方法**：`Config.get(key, default=None)`
- **实现**：按 3.2 节伪代码解析点号路径，返回对应值或默认值。

## 7. 深拷贝保护
- **方法**：`Config.to_dict()`
- **职责**：返回 `copy.deepcopy(self._config)`，防止外部直接修改内部配置。

## 8. 与原有逻辑的兼容性
- 原有 `GameState._load_config()` 中的配置加载逻辑将被抽取到 `Config` 类中。
- `GameState` 持有 `Config` 实例，通过 `self.config.get()` 访问配置。
